# SaaSto LP UI Test (Portfolio 2026)

SaaS ランディングページ（6画面）の UI テストを題材にした、ポートフォリオ用リポジトリです。  
**「UI モックしかない状態から、テスト計画 → テスト設計 → テストケース → 不具合起票までを一人で回せる」ことを示す**のが主目的です。
付録的に、Playwright を使ったリンク遷移テストのサンプルコードも 1 本だけ含めています。

---

## 1. テスト対象

Figma などで作成された LP モック画像 6枚をテスト対象としました。

- PC 版
  - `Home_01`（SNS / リーチ系）
  - `Home_02`（Time Tracking アプリ）
  - `Home_03`（Finance アプリ）
- SP 版
  - `Mobile-Home_01`
  - `Mobile-Home_02`
  - `Mobile-Home_03`

（※ 画面デザインそのものは Figma の UI テンプレートを「モック」として利用しており、このリポジトリの主題はあくまでテスト設計・ドキュメンテーションになります）
Figma の UI テンプレートの参照元:  
[Community File](https://www.figma.com/community/file/1202426685198179521)
---

## 2. テストの目的

- デザインモック通りに **UI・文言・リンク** が実装されていることを確認する  
- 代表的なデバイス幅でレイアウトが崩れず、**CTA（ボタン・リンク）が正常に動作**することを確認する  
- LP レベルの画面に対して、短時間（4〜5h 程度）で **テスト計画〜実行までを完結できるプロセス**を示す  

---

## 3. 想定環境

最低限、以下の環境を対象としました。

- PC：  
  - **どちらか 1 環境を選択**
    - Windows / Chrome 最新
    - Mac / Safari 最新  
  - ※ Linux はユーザー比率が低いためサポート対象外

- SP：  
  - iOS / Safari  
  - Android / Chrome  

代表解像度：

- PC：1440px
- SP：375px（縦は MUST、横画面は NICE）

## 5. テスト方針とスコープ

### 5.1 テスト種類

4時間に収める想定で、MUST / NICE を分離しています。

- 機能テスト
  - ヘッダーナビ（Home / About us / Services / Blog / Contact us）
  - Login / Sign up リンク
  - Hero セクションの CTA（Get Started / Start 14 Days Trial）
  - SP のハンバーガーメニュー開閉
- UI / レイアウトテスト
  - モックとの整合性（フォント・色・余白・角丸・アイコン位置）
  - 共通コンポーネント（実績カード、CTA ボタン）のスタイル統一
  - 画像やグラフカードの位置関係・崩れの有無
- レスポンシブテスト
  - 1440px / 375px でのレイアウト切り替え
  - 2カラム → 1カラムの変化
  - 数値カードの折り返し・縦並び
- コンテンツ / 文言テスト
  - メインコピー 3種の綴り・文法
  - 数値表記（15k+ / 30k / 10k / >10k）
  - ラベルの一貫性（Customer / Regular Customer など）
- アクセシビリティ（ライト版）
  - img / ロゴへの alt 付与
  - Tab / Enter で CTA が操作可能か
  - （NICE）コントラスト比・見出し階層構造

### 5.2 スコープ外

- パフォーマンス計測（LCP / CLS など）
- Edge / Firefox など全ブラウザ網羅
- バックエンド連携（フォーム送信・メール送信・決済処理）

## 6. 付録：テスト自動化サンプル（Playwright）

このリポジトリでは、**テスト設計（手動テスト）がメイン**です。
おまけとして、Playwright によるリンク遷移テストのサンプルを 1 本だけ置いています。

### 6.1 ファイル

- `tests/saasto-links.spec.ts`  
  - ヘッダーナビ + CTA のリンク遷移を確認
  - 例外時に **どのステップで失敗したか分かるロギング (`runStep`)** 付き

確認している観点：

- PC 幅でのヘッダーナビリンク
- Login / Sign up への遷移
- Hero セクション CTA（Get Started Now）からの遷移
