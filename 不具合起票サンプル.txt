[Bug] 「Good Tesimonials」のスペルミス #001
■概要
Home_03 の実績カード文言「Good Tesimonials」にスペルミスがある。
正しくは「Good Testimonials」と表記するのが自然と思われる。

■発生箇所
- Home_03（PC / SP 共通想定）
- Hero セクション下部の実績カード（`>10k Good Tesimonials` のカード）

■現象
- 実績カードのラベルが「Good Tesimonials」と誤記されている。
- 「Testimonials」の「n」が抜けており、英語として不自然な表記になっている。

■期待される挙動
- 実績カードのラベルは「Good Testimonials」と正しい綴りで表示されること。
- 他言語にローカライズする場合も、元の英語はスペルミスのない状態を基準とする。

■再現手順
1. PC もしくは SP で Home_03 を表示する。
2. ページ下部の実績値カード群を確認する。
3. `>10k` の数値が表示されているカードのラベルを確認する。
4. テキストが「Good Tesimonials」と表示されていることを確認する。

■再現回数
5/5

■補足
- デザインモック、実装の両方で同じ誤記がある可能性があるため、元デザインの修正も要検討。
- テキストはコンポーネント化している場合、同じラベルが他ページにも再利用されていないか確認が必要。

■影響度
- 低〜中
- UX/ブランディング面での信頼性低下につながる可能性があるため、優先度は低くないが致命的ではない。

[Bug] 「Customer」と「Regular Customer」の表記揺れ #002
■概要
実績カードの3枚目のラベルが、画面によって「Customer」と「Regular Customer」で表記揺れしている。
仕様として明確な意図が無い場合、どちらかに統一した方がユーザーにとって分かりやすい。

■発生箇所
- Home_01 / Home_02 / Home_03（PC / SP）
- ページ下部の実績値カード（3枚目のカード）

■現象
- ある画面ではラベルが「Customer」と単数形で表示されている。
- 別の画面ではラベルが「Regular Customer」と表示されており、意味・ニュアンスが異なる。
- 「Regular」が付く意図（リピーター／継続顧客など）が仕様上明示されていないため、表記揺れに見える状態。

■期待される挙動
- 仕様上の意図に基づき、以下のいずれかに統一されていること。
  - すべて「Customer」に統一する
  - すべて「Regular Customer」に統一する
- もし指標として別物（顧客数 vs 継続顧客数）を表しているなら、
  - ラベルや補足テキストで違いが分かるようにする（例：`Customer` / `Regular Customer (repeat)` など）

■再現手順
1. Home_01（PC）を表示し、実績カード3枚目のラベルを確認する。
2. Mobile-Home_01 を表示し、同じ位置のカードラベルを確認する。
3. Home_02 / Home_03 についても、PC / SP で3枚目のカードラベルを比較する。
4. 画面によって「Customer」「Regular Customer」が混在していることを確認する。

■再現回数
5/5

■補足
- デザイン段階で意図的に変えている可能性もあるため、プロダクトオーナーへの意図確認が必要。
- KPI 用語として「Customer」「Active user」との関係性も整理しておくと、今後の画面追加時の表記ルールにも活かせる。

■影響度
- 低〜中
- 直接の機能障害ではないが、メトリクスの意味が曖昧になると信頼性や説明コストに影響する。

[Bug] スマホ幅でハンバーガーメニューが開かない #003
■概要
SP 表示時にヘッダー右上のハンバーガーメニューをタップしても、ナビゲーションメニューが開かない。
SP でページ内移動ができず、実質的にヘッダーナビが機能していない状態。

■発生箇所
- Mobile-Home_01 / Mobile-Home_02 / Mobile-Home_03
- 画面上部ヘッダーのハンバーガーメニューアイコン

■現象
- SP 幅（375px相当）でページを表示。
- ヘッダー右上のハンバーガーアイコンをタップしても、ドロワーメニューが表示されない。
- アイコンはタップ反応（押下エフェクト）はあるが、メニュー表示領域が出てこない。
- PC 版ではヘッダー内のメニューリンクが正常に表示されているため、SP 特有の問題と思われる。

■期待される挙動
- SP 幅でハンバーガーアイコンをタップすると、画面右（または左）からナビゲーションメニューがスライドインして表示されること。
- メニュー表示中に再度アイコンや背景をタップすると、メニューが閉じること。
- 各メニュー項目タップで該当セクション／ページへ遷移し、メニューが閉じること。

■再現手順
1. スマートフォン端末、またはブラウザの開発者ツールで幅 375px の縦画面に設定する。
2. Mobile-Home_01 を表示する。
3. 画面上部のハンバーガーアイコンをタップする。
4. ナビゲーションメニューが表示されないことを確認する。
5. Mobile-Home_02 / 03 でも同様に確認し、いずれもメニューが開かないことを確認する。

■再現回数
5/5

■補足
- JavaScript の初期化漏れ、もしくは SP 向けのナビコンポーネントの読み込み条件が誤っている可能性がある。
- コンソールエラーの有無、PC 幅→SP 幅にリサイズした場合の挙動（動的切り替え）も合わせて確認すると切り分けしやすい。

■影響度
- 中〜高
- SP 利用ユーザーがヘッダーナビを利用できないため、ページ内導線の多くが失われる。
- LP 起点のコンバージョンフロー次第では、CV率に直接影響する可能性がある。
